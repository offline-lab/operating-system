# Schemas

Machine-readable schemas for all Offline Lab data formats. Implementors of appctl,
buildctl, or compatible tooling should validate against these.

---

## Package formats

| Schema | Format | Description |
|---|---|---|
| [package-yaml.schema.json](schemas/package-yaml.schema.json) | JSON Schema | `package.yaml`: build-time package definition written by app authors |
| [package-metadata.schema.json](schemas/package-metadata.schema.json) | JSON Schema | `<name>-<version>-<arch>.json`: metadata generated by buildctl, shipped with the squashfs |

See [Package Format](specs/package-format.md) for the full field reference and examples.

---

## Repository formats

| Schema | Format | Description |
|---|---|---|
| [repo-index-root.schema.json](schemas/repo-index-root.schema.json) | JSON Schema | Root `index.json`: discovery document listing signing keys and per-arch index URLs |
| [repo-index-arch.schema.json](schemas/repo-index-arch.schema.json) | JSON Schema | `packages/<arch>/index.json`: per-arch package index, one entry per package (latest version) |

See [Repository](specs/repository.md) for the directory layout, signing model, and client flow.

---

## On-device database

| Schema | Format | Description |
|---|---|---|
| [packages.sql](schemas/packages.sql) | SQLite DDL | `packages.db`: appctl's on-device state database (repos, cached index, installed packages, retained images) |

---

## Host resource baseline

| Schema | Format | Description |
|---|---|---|
| [resources.schema.json](schemas/resources.schema.json) | JSON Schema | `/data/config/resources.json`: host resource baseline written at boot by `offlinelab-resources.service` |

See [Resource Tracking](specs/resource-tracking.md) for how appctl uses this file for pre-install capacity checks.
